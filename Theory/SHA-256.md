SHA-256 (Secure Hash Algorithm 256-bit) is a cryptographic hash function that’s part of the SHA-2 family, designed by National Security Agency.

In simple terms, it takes any input (string, file, password, etc.) and converts it into a fixed 256-bit (32-byte) hash value, usually shown as a 64-character hexadecimal string.

🔑 Key properties of SHA-256
Deterministic → same input always gives same hash
Fixed length output → always 256 bits, no matter input size
One-way function → you cannot reverse it to get original data
Collision resistant → extremely hard to find two inputs with same hash
Sensitive to change → even 1 character change → completely different hash

Example:

Input:  "hello"
Hash:   2cf24dba5fb0a... (64 hex chars)

Input:  "hello1"
Hash:   totally different output
💡 Why SHA-256 is used in crypto packages

In programming (like Node.js crypto module or Java MessageDigest), SHA-256 is used for:

1. Password hashing

You don’t store passwords directly:

password → SHA-256 → store hash

⚠️ In real systems, you combine it with salt + stronger algorithms (like bcrypt)

2. Data integrity verification

To ensure data isn’t tampered:

original data → hash
received data → hash → compare
3. Digital signatures

Used along with encryption to verify authenticity of messages

4. Blockchain (big one 🚀)

Used in Bitcoin for:

Mining
Block hashing
Transaction security
🧑‍💻 Example (Node.js)
import crypto from "crypto";

const hash = crypto
  .createHash("sha256")
  .update("hello")
  .digest("hex");

console.log(hash);
⚠️ Important note

SHA-256 alone is fast, so it's not ideal for password storage by itself (because fast = easier to brute force). Use:

bcrypt
scrypt
argon2
